## 1. 启动方式

### 1.1 直接运行（推荐给非开发用户）

1. 进入 `dist` 目录，双击运行 `AI_Novel_Writer.exe`
2. 程序会自动启动本地服务并打开浏览器页面

说明：
- 首次启动可能需要几十秒（onefile 会先解压依赖）
- 如果浏览器没有自动弹出，可以稍等 10-30 秒后手动打开页面（一般为 `http://localhost:8501` 起的某个端口）


## 2. API 配置（首次必做）

1. 打开页面左侧边栏
2. 展开「系统设置」
3. 点击「API & 授权」
4. 填入各平台 Key，并点击「保存配置并验证」

### 2.1 需要配置哪些 Key

- DeepSeek API Key：用于「大纲动态优化师」
- 通义千问（DashScope）API Key：用于主写、角色一致性、润色、Embedding（记忆）
- Kimi API Key：可选；在某些策略下用于主写/终审，提高文风但通常更慢

### 2.2 模型链路策略与主写模型

- 模型链路策略（ROUTE_PROFILE）
  - speed：全链路优先速度
  - balanced：部分环节倾向更高质量（若 Kimi 可用）
  - quality：更偏质量（若 Kimi 可用）
- 主写模型（WRITER_MODEL）
  - auto：按策略自动选择
  - qwen：更稳定更快
  - kimi：文风潜力更高但通常更慢（需要 Kimi Key）

### 2.3 验证连接

点击「保存配置并验证」后，会逐项显示：
- ✅ 连接成功：可以开始生成
- ❌ 连接失败：通常是 Key 无效、网络受限、或服务端限流

提示：
- 连接失败也会保存配置，你可以稍后更换 Key 再验证

### 2.4 配置保存位置

程序会在运行目录保存两类文件（不要随意删除）：
- `.api_keys.enc`：加密后的配置
- `.encryption_key`：本机加密密钥

如果你复制了 exe 到新目录运行，这两份文件也会在新目录重新生成（配置互不影响）。

## 3. 创建项目与填写大纲

### 3.1 新建项目

1. 左侧边栏「我的项目」
2. 点击「新建」
3. 输入小说名称并创建

创建成功后会自动生成项目目录（位于 `projects/<项目名>/`），包括：
- `大纲.md`
- `chapters/`（章节）
- `summaries/`（摘要）
- `logs/`（运行日志）
- `剧情圣经.md`（系统根据大纲提炼，用于一致性）

### 3.2 填写并保存大纲

1. 进入「创作中心」
2. 在「故事大纲」的「编辑」页粘贴/编写大纲
3. 点击「保存大纲」

建议：
- 大纲至少 50 字才能启动生成
- 使用 `第N章` 格式能更好识别章节数量
- 前 800-1200 字尽量包含：世界观硬约束、人物小传、关键矛盾、成长路径

## 4. 生成章节

1. 在右侧「智能生成」面板选择模式
   - 智能续写：从已生成的最后一章继续
   - 批量生成：指定起始章节与数量
2. 点击「启动生成引擎」
3. 等待生成完成

提示：
- 生成时可能需要几分钟，这是正常的（多 Agent 协作）
- 如长时间无响应，可查看「运行监控」标签页或程序日志

## 5. 阅读管理

在「阅读管理」中：
- 左侧选择章节（支持搜索、上一章/下一章）
- 右侧阅读正文

## 6. 导出发布

在「导出发布」中可导出：
- TXT
- Word（docx）
- EPUB

导出文件默认会生成在当前工作目录或指定导出目录中。

## 7. 常见问题排查

### 7.1 页面一直在加载

常见原因：
- 首次启动解压较慢
- 端口被占用
- 后端启动过程中异常但没有弹出（windowed 模式无控制台）

建议做法：
- 等待 30-60 秒
- 关闭占用端口的软件（或重启电脑）
- 查看 exe 同目录生成的 `ai_novel_startup.log`（若有）

### 7.2 打包失败或提示“拒绝访问”

常见原因：
- `dist\\AI_Novel_Writer.exe` 正在运行，导致无法覆盖写入

解决：
- 先关闭正在运行的程序再打包

### 7.3 API 验证失败

常见原因：
- Key 填错/过期
- 网络环境无法访问对应服务
- 短时间内请求过多触发限流

解决：
- 更换 Key 后重新验证
- 稍等 1-5 分钟再试
- 检查网络代理/防火墙
