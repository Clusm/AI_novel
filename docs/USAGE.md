# 使用指南（USAGE）

本指南面向使用者，覆盖安装运行、API 配置、创建项目、大纲写法、生成章节、导出与排错。

## 运行方式

### 方式 A：直接运行 EXE（非开发用户推荐）

1. 进入 `dist/` 目录
2. 双击运行 `AI_Novel_Writer.exe`
3. 程序会启动本地服务并自动打开浏览器页面

说明：

- 首次启动可能需要几十秒（onefile 会先解压依赖）
- 如果浏览器未自动弹出，可稍等 10–30 秒后手动打开（通常从 `http://localhost:8501` 起的某个端口）

### 方式 B：从源码运行（开发/调试）

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip
pip install -e .
streamlit run app.py
```

## API 配置（首次必做）

1. 打开页面左侧边栏
2. 展开「系统设置」
3. 点击「API & 授权」
4. 填入各平台 Key，并点击「保存配置并验证」

### 需要配置哪些 Key

- DeepSeek API Key：用于大纲/推理等环节（取决于链路策略）
- 通义千问（DashScope）API Key：用于主写、设定守护、Embedding（记忆）等
- Kimi API Key：可选；在部分策略下用于主写/终审，提高文风但可能更慢

### 配置保存位置

程序会在运行目录保存两类文件：

- `.api_keys.enc`：加密后的配置
- `.encryption_key`：本机加密密钥

不要随意删除，也不要提交到公共仓库。

## 创建项目与填写大纲

### 新建项目

1. 左侧边栏「我的项目」
2. 点击「新建」
3. 输入小说名称并创建

项目资产会生成在 `projects/<项目名>/`，包括：

- `大纲.md`
- `chapters/`（章节正文）
- `summaries/`（摘要）
- `canon/`（事实台账）
- `logs/`（运行日志）
- `剧情圣经.md`（从大纲提炼，用于一致性）

### 大纲写法建议（影响可控与一致性）

- 开头 800–1200 字：集中写“硬约束”（世界观铁律/人设底线/节奏铁律/禁忌）
- 章节细纲尽量使用 `第N章` 作为标题行，便于系统按章抽取相关片段

参考：[大纲模板（参考）.md](file:///c:/Users/Tao/Documents/trae_projects/AI_novel/大纲模板（参考）.md)

## 生成章节

1. 在右侧「智能生成」面板选择模式
   - 智能续写：从已生成的最后一章继续
   - 批量生成：指定起始章节与数量
2. 点击「启动生成引擎」
3. 等待生成完成

### 生成结果的规范

- 最终正文会强制满足长度下限（默认 ≥3500 字），不足会自动扩写补足
- 终审定稿会自动去除章节内情节标题（只保留 `# 第N章 标题`）
- 写手会在正文末尾附加摘要块，但会在终审/保存阶段剥离并单独保存到 `summaries/`

## 阅读管理

在「阅读管理」中：

- 左侧选择章节（支持搜索、上一章/下一章）
- 右侧阅读正文

## 导出发布

在「导出发布」中可导出：

- TXT
- Word（docx）
- EPUB

## 常见问题

更完整的排错见：[FAQ.md](file:///c:/Users/Tao/Documents/trae_projects/AI_novel/docs/FAQ.md)

- 页面一直在加载：首次解压慢/端口占用/后端异常。等待 30–60 秒或查看同目录 `ai_novel_startup.log`
- API 验证失败：Key 失效、网络受限或限流。更换 Key 或稍后重试
- EXE 打包失败提示“拒绝访问”：通常是 `dist/AI_Novel_Writer.exe` 正在运行，先关闭再打包
