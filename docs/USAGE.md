# 使用指南（USAGE）

本指南面向使用者，覆盖安装运行、API 配置、创建项目、大纲写法、生成章节、阅读管理与导出。

## 1. 安装与运行

### 方式 A：使用安装包（推荐）

1. 获取最新的安装包 `AI_Novel_Writer_Setup_vX.X.exe`。
2. 双击安装，按照提示完成安装。
3. 从桌面快捷方式或开始菜单启动 **AI_Novel_Writer**。

### 方式 B：使用绿色版 (Portable)

1. 获取 `dist/AI_Novel_Writer` 文件夹。
2. 双击文件夹内的 `AI_Novel_Writer.exe` 即可直接运行。
3. 数据将存储在您的 `Documents/AI_Novel_Projects` 和 `%APPDATA%/AI_Novel_Writer` 目录下，方便迁移。

### 方式 C：源码运行（开发/调试）

```powershell
# 1. 创建虚拟环境
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# 2. 安装依赖
pip install -e .

# 3. 启动 GUI
python run_app.py
```

## 2. 系统初始化配置

首次启动后，请点击左侧边栏底部的 **「系统设置」** 图标（齿轮形状）：

1. **API 授权**：
   - **DeepSeek API Key**：用于大纲分析、情节细化（推荐）。
   - **DashScope (通义千问)**：用于长文本生成、角色扮演、记忆检索（必须）。
   - **Moonshot (Kimi)**：用于润色和优化文风（可选）。
   - 点击 **「保存配置并验证」**，系统会自动测试 Key 的有效性。

2. **模型链路选择**：
   - **Speed (极速模式)**：DeepSeek-V3 + Qwen-Plus，成本最低，速度最快。
   - **Balanced (平衡模式)**：DeepSeek-V3 + Qwen-Max/Kimi，平衡质量与速度（推荐）。
   - **Quality (深度模式)**：DeepSeek-R1 + Qwen-Max，逻辑最强，但速度较慢。

### 2.2 模型组合策略详解

系统会根据你选择的 **路由策略** 和 **主写模型** 自动组合不同的 AI 模型，以达到最佳效果：

| 路由策略 | 大纲优化 (DeepSeek) | 设定守护 (Qwen) | 主写手 (Writer) | 润色审校 (Editor) | 特点 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Speed (极速)** | deepseek-chat | qwen-plus | qwen-plus | qwen-plus | **速度最快**，成本最低，适合快速出稿。 |
| **Balanced (平衡)** | deepseek-chat | qwen-plus | qwen-plus | **kimi-k2.5** | 兼顾速度，**润色环节**使用 Kimi 提升文笔。 |
| **Quality (质量)** | **deepseek-reasoner (R1)** | **qwen-max** | **kimi-k2.5** | **kimi-k2.5** | **质量最优**，使用深度思考模型和大参数模型，文笔细腻，但生成较慢。 |

> **注意**：
> *   如果未配置 Kimi Key，系统会自动回退使用 Qwen 模型。
> *   DeepSeek R1 (reasoner) 模型思考时间较长，请耐心等待。

## 3. 项目管理

### 新建项目
1. 点击左侧边栏的 **「新建项目」**。
2. 输入小说名称（如《我的AI写作生涯》）。
3. 在弹出的编辑器中输入或粘贴您的**小说大纲**。
   - *提示：大纲越详细，生成的连贯性越好。建议包含【世界观】、【核心角色】、【分卷细纲】。*
   - 参考：[大纲模板 (outline_template.md)](outline_template.md)

### 项目结构
您的项目数据默认存储在 `Documents/AI_Novel_Projects/<项目名>`：
- `大纲.md`：您输入的原始大纲。
- `chapters/`：生成的章节正文。
- `summaries/`：AI 自动生成的剧情摘要（用于上下文记忆）。
- `canon/`：事实台账（记录物品、等级、关系变化）。
- `剧情圣经.md`：系统自动维护的全局设定集。

## 4. 创作中心

点击左侧边栏的 **「创作中心」** 进入核心写作界面：

1. **当前状态**：显示当前已生成的章节数、总字数。
2. **生成控制**：
   - **目标章节**：选择要生成的章节号（系统会自动识别大纲中的 `第N章`）。
   - **生成数量**：一次性生成多少章（建议 1-3 章）。
3. **启动引擎**：点击 **「启动生成引擎」**，系统将调度 AI Agent 开始工作：
   - **Planner**：拆解大纲，设计爽点。
   - **Guardian**：检查人设，防止OOC。
   - **Writer**：撰写正文（3000+字）。
   - **Reviewer**：润色修辞，填补逻辑漏洞。

*注意：生成过程中请保持网络通畅，单章生成约需 2-5 分钟。*

## 5. 阅读与导出

### 阅读管理
- 点击 **「阅读管理」** 查看已生成的章节。
- 支持 **全文搜索**、**按章跳转**。
- **一键复制**：点击正文右上角的复制按钮，即可将整章内容复制到剪贴板。

### 导出发布
点击 **「导出发布」**，支持多种格式：
- **TXT**：纯文本打包。
- **Word (.docx)**：带排版的文档。
- **EPUB**：电子书格式。
- **Markdown**：原始格式。
