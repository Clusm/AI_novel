# 大纲模板（参考，适配当前工作流）

该模板面向本项目的生成流水线：先由系统从大纲提炼《剧情圣经》，再按章走“提纲→人设/世界观→爽点→正文→审查→润色”。  
为了让每章生成更稳定，建议把“必须每章都遵守的硬约束”尽量放在文档最前面 800–1200 字内。

重要：每章任务注入的大纲不是“永远取开头”，而是会拼接两部分内容并控制总长度：

- 大纲开头的一段“硬约束摘要”
- 当前章节号对应的“章节相关片段”（优先匹配 `第N章` 标题段落）

---
